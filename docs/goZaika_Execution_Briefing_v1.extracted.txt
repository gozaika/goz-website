I.1  A.1 — Logo Usage Matrix (Plain Text)
Horizontal Colour (gozaika-logo-color.svg): Desktop nav header. Email headers. Pitch decks. Press kit. Letterhead. Minimum width 140px. Aspect ratio 600:153 — never distort. Do not use below 100px or as an app icon.
Horizontal White (gozaika-logo-white.svg): Any dark background. Dark nav bar. Dark footer. Dark-mode OG images. Email footer on dark bg. Minimum width 140px. Do not use on light backgrounds — it will be invisible.
Symbol Only (gozaika-app-icon.svg): App icon (iOS, Android). Favicon 32px+. Social media avatar. OG image corner mark. Loading spinner. WhatsApp group icon. Minimum 32px. This is the only variant that works below 100px.
Colour on Dark: No separate file needed. Use the white wordmark on the Forest Green (#1A5C38) background — this is the header state.
Prohibited in all variants: Stretching or squashing the aspect ratio. Adding drop shadows. Placing on busy photo backgrounds without a solid colour panel behind it. Rotating. Changing any colour value. Adding taglines or additional text inside the logo lockup.

Hero headline: Playfair Display weight 700. Scale 6xl (60px desktop, 42px mobile). Line-height 1.05. Letter-spacing -0.05em. Colour: gray900 (#111827). Maximum 2 lines. Used only on the homepage hero.
Hero sub-headline: Poppins weight 400. Scale xl (20px). Line-height 1.4. Letter-spacing -0.015em. Colour: forest (#1A5C38). Used only on homepage hero below the main headline.

Card headings H3: Poppins weight 600. Scale 2xl (24px). Line-height 1.35. Letter-spacing -0.02em. Colour: gray900.
Body text: Poppins weight 400. Scale base (16px). Line-height 1.625. Letter-spacing 0em. Colour: gray700 (#374151). Maximum line length: 680px (72ch) for optimal readability.
Navigation items: Poppins weight 500. Scale sm (14px). Line-height 1.5. Colour: white on dark header. No uppercase.

Badge and tag text: Poppins weight 600. Scale xs (12px). Letter-spacing 0.08em. Uppercase: YES. Used for cuisine tags, allergen badges, status labels.
Form labels: Poppins weight 500. Scale sm (14px). Colour: gray700. Always visible above the input — never inside as placeholder-only.
Legal body text: Poppins weight 400. Scale sm (14px). Line-height 1.7. Colour: gray500. Used only on /privacy-policy, /terms-of-service, /refund-policy, /food-safety-policy, /grievance-redressal.
Font weight restriction: Use ONLY weights 400, 500, 600. Weight 700 only in Playfair Display. Never bold (700) Poppins in UI — it reads as shouting. Weight 300 (light) is never used.

I.10  G.2 — Accessibility Protocol (Explicit Checklist)
Run in this order: automated first, then keyboard, then screen reader. All must pass before launch.
STEP 1: Automated (axe DevTools browser extension)
Install axe DevTools (free) from Chrome Web Store
Run on every page. Zero critical violations. Zero serious violations.
Known false positives: ignore 'region' landmark warnings if your page structure is correct
STEP 2: Colour Contrast Manual Check
White text on Forest Green (#1A5C38): ratio 8.9:1 — PASS
White text on Saffron (#FF6B35): ratio 3.0:1 — FAIL for body text. Use only for large text (>18px) or bold large text (>14px bold). Never use white on saffron for body copy or form labels.
Gray700 (#374151) on White: ratio 10.1:1 — PASS
Gray500 (#6B7280) on White: ratio 4.6:1 — PASS (just above 4.5:1 minimum)
Saffron (#FF6B35) on Cream (#FFF8F0): ratio 2.9:1 — FAIL. Never use orange text on cream background.
Forest Green (#1A5C38) on Cream (#FFF8F0): ratio 8.4:1 — PASS
STEP 3: Keyboard Navigation Test
Press Tab from the browser address bar. Every interactive element (links, buttons, inputs) must receive a visible focus ring in sequence.
Focus order must match visual reading order (left-to-right, top-to-bottom).
Mobile hamburger menu: Tab opens it. Escape closes it. Focus returns to hamburger button.
Accordion items (FAQ): Enter or Space opens/closes. Arrow keys navigate between items.
Forms: Tab moves through all fields. Enter submits. Escape does not clear filled form.
Modals (if any): Focus must be trapped inside. Escape closes. Focus returns to trigger element.
STEP 4: Screen Reader Test
Mac (VoiceOver): Cmd+F5 to enable. Navigate with VO+Arrow keys. Every section must have a logical heading hierarchy (H1 → H2 → H3, no skips).
Test: hero section reads headline, sub-headline, CTA label in logical order.
Test: nav reads 'goZaika logo, link' then each nav item label.
Test: form reads each label before its input. Required fields read 'required'.
Test: error messages are announced immediately when validation fails (use aria-live='polite').
Test: accordion reads 'Question text, button, expanded/collapsed'.
STEP 5: Skip-to-Content Link
A hidden <a href='#main-content'>Skip to main content</a> must be the very first element in <body>.
It becomes visible only on Tab focus (use className='sr-only focus:not-sr-only focus:absolute ...')
The main content area must have id='main-content' and tabIndex={-1}.

V.3  Row Level Security (RLS)
Enable RLS on all three tables. The website only inserts, never reads. Only you (authenticated admin) can read.
-- In Supabase SQL Editor:
ALTER TABLE waitlist_leads ENABLE ROW LEVEL SECURITY;
ALTER TABLE contact_submissions ENABLE ROW LEVEL SECURITY;
ALTER TABLE partner_interests ENABLE ROW LEVEL SECURITY;
-- Allow anonymous inserts (website)
CREATE POLICY 'anon_insert_waitlist' ON waitlist_leads
FOR INSERT TO anon WITH CHECK (true);
CREATE POLICY 'anon_insert_contact' ON contact_submissions
FOR INSERT TO anon WITH CHECK (true);
CREATE POLICY 'anon_insert_partner' ON partner_interests
FOR INSERT TO anon WITH CHECK (true);
-- Only authenticated users can SELECT (your admin sessions)
CREATE POLICY 'auth_select_waitlist' ON waitlist_leads
FOR SELECT TO authenticated USING (true);
ℹ  With these policies: the website can INSERT using the anon key. Nobody can SELECT without logging into Supabase. Your leads are safe.

import { supabase } from '@/lib/supabase'
import { Resend } from 'resend'
const resend = new Resend(process.env.RESEND_API_KEY)
export async function POST(req: Request) {
const { name, email, city, role } = await req.json()
// 1. Save to Supabase
const { error } = await supabase.from('waitlist_leads')
.insert({ name, email, city, role })
if (error && error.code !== '23505') {  // 23505 = duplicate email
return Response.json({ ok: false, error: 'Database error' }, { status: 500 })
}
// 2. Send notification email (non-blocking)
await resend.emails.send({
from: 'goZaika <noreply@resend.dev>',  // update after domain
to: 'waitlist@gozaika.in',
subject: `New waitlist signup: ${name} (${city})`,
text: `Name: ${name}\nEmail: ${email}\nCity: ${city}\nRole: ${role}`,
})
return Response.json({ ok: true })
}
