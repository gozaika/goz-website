A.2.5  Motion Tokens — HARD CONSTRAINTS
These are non-negotiable. Any animation outside these bounds fails QA.
export const motion = {
// Durations
instant:  100,   // ms — tooltips, toggles
fast:     150,   // ms — hover states, button feedback
normal:   250,   // ms — most UI transitions
slow:     400,   // ms — page-level entrances, modals
max:      400,   // ms — HARD CAP. Nothing longer on element level.
// Easings (as Framer Motion objects)
standard: [0.2, 0, 0, 1],          // Material You standard curve
decelerate: [0, 0, 0.2, 1],         // Entrances (things sliding in)
accelerate: [0.4, 0, 1, 1],         // Exits (things sliding out)
// Viewport trigger
scrollTrigger: { once: true, amount: 0.15 },  // fire once at 15% in view
stagger: 0.08,  // seconds between staggered children
};
MOTION RULES — NON-NEGOTIABLE
NO parallax effects — ever. They cause motion sickness and hurt CLS.
NO autoplay animations — user must scroll to trigger all entrances.
NO duration > 400ms on any individual element.
YES prefers-reduced-motion — wrap all Framer Motion in useReducedMotion hook.
YES opacity-only on mobile — skip translate animations on < 768px for perf.
YES test on real Android mid-range device (Samsung A-series) before launch.
