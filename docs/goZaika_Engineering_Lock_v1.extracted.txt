Part 2 — Rate Limiting: Full Implementation
DECISION: Upstash Redis (free tier) for rate limiting
Why not in-memory Map: Vercel runs serverless functions — each invocation is a fresh
process. An in-memory Map resets on every cold start and is meaningless for rate limiting.
Why Upstash: Redis-as-a-service. Free tier: 10,000 commands/day (~8K requests).
Specifically designed for Vercel Edge + serverless. First-class Next.js support.
Zero infrastructure. Works on free Vercel plan. Setup: 5 minutes.
Rate limits applied: All three public API routes (/api/waitlist, /api/contact,
/api/partner-interest). Limits: 5 requests per IP per 15 minutes per endpoint.

2.2  Rate Limiter Module
Create lib/ratelimit.ts — one file, used by all API routes:
// lib/ratelimit.ts
import { Ratelimit } from '@upstash/ratelimit'
import { Redis } from '@upstash/redis'
import { NextRequest } from 'next/server'
const redis = new Redis({
url:   process.env.UPSTASH_REDIS_REST_URL!,
token: process.env.UPSTASH_REDIS_REST_TOKEN!,
})
// 5 requests per 15 minutes, sliding window
const limiter = new Ratelimit({
redis,
limiter: Ratelimit.slidingWindow(5, '15 m'),
analytics: true,   // visible in Upstash dashboard
prefix: 'gozaika', // namespace keys in Redis
})
/**
* Returns null if allowed, or a Response with 429 if blocked.
* Usage:
*   const block = await checkRateLimit(req, 'waitlist')
*   if (block) return block
*/
export async function checkRateLimit(
req: NextRequest,
identifier: string
): Promise<Response | null> {
const ip =
req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() ??
req.headers.get('x-real-ip') ??
'127.0.0.1'
const key = `${identifier}:${ip}`
const { success, limit, remaining, reset } = await limiter.limit(key)
if (!success) {
const retryAfter = Math.ceil((reset - Date.now()) / 1000)
return new Response(
JSON.stringify({
ok: false,
error: 'Too many requests. Please try again later.',
}),
{
status: 429,
headers: {
'Content-Type': 'application/json',
'X-RateLimit-Limit':     String(limit),
'X-RateLimit-Remaining': String(remaining),
'Retry-After':           String(retryAfter),
},
}
)
}
return null
}
2.3  Integration into API Routes
All three public API routes follow this exact pattern:
// app/api/waitlist/route.ts
import { NextRequest } from 'next/server'
import { checkRateLimit } from '@/lib/ratelimit'
import { supabase } from '@/lib/supabase'
import { Resend } from 'resend'
const resend = new Resend(process.env.RESEND_API_KEY)
export async function POST(req: NextRequest) {
// ① Rate limit check — must be first
const block = await checkRateLimit(req, 'waitlist')
if (block) return block
// ② Parse + validate body
let body: unknown
try { body = await req.json() }
catch { return Response.json({ ok: false, error: 'Invalid JSON' }, { status: 400 }) }
const { name, email, city, role } = body as Record<string, string>
if (!name?.trim() || !email?.trim() || !city?.trim() || !role?.trim())
return Response.json({ ok: false, error: 'All fields required' }, { status: 400 })
if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email))
return Response.json({ ok: false, error: 'Invalid email' }, { status: 400 })
if (!['consumer', 'restaurant'].includes(role))
return Response.json({ ok: false, error: 'Invalid role' }, { status: 400 })
// ③ Persist to Supabase
const { error: dbErr } = await supabase
.from('waitlist_leads')
.insert({ name: name.trim(), email: email.toLowerCase().trim(), city, role })
if (dbErr && dbErr.code !== '23505')  // 23505 = unique violation (already on list)
return Response.json({ ok: false, error: 'Database error' }, { status: 500 })
// ④ Notify via Resend (non-blocking — fire and forget)
void resend.emails.send({
from: 'goZaika Waitlist <noreply@resend.dev>',
to:   'waitlist@gozaika.in',
subject: `Waitlist: ${name} (${city} · ${role})`,
text:    `Name: ${name}\nEmail: ${email}\nCity: ${city}\nRole: ${role}`,
})
return Response.json({ ok: true })
}
2.4  Input Sanitisation
All text fields are sanitised before DB insert. Add this to lib/sanitize.ts:
// lib/sanitize.ts
/**
* Strip HTML tags and trim whitespace.
* Use on ALL string inputs before DB insert.
*/
export function sanitize(value: unknown, maxLength = 500): string {
if (typeof value !== 'string') return ''
return value
.replace(/<[^>]*>/g, '')  // strip HTML
.replace(/[\u0000-\u001F]/g, '')  // strip control chars
.trim()
.slice(0, maxLength)
}
// Usage in API route:
import { sanitize } from '@/lib/sanitize'
const cleanName    = sanitize(name, 100)
const cleanMessage = sanitize(message, 2000)

Part 3 — Bot & Spam Mitigation
DECISION: Cloudflare Turnstile on all three public forms
Why Turnstile over hCaptcha, reCAPTCHA v2, or reCAPTCHA v3:
Turnstile:    Free (no tiers). Non-intrusive (no puzzle). GDPR/DPDP-safe (no
fingerprinting). Works invisibly for humans. Blocks bots effectively.
Native Cloudflare CDN integration — low latency for Indian users.
Simple React integration: one component, one server-side verify call.
hCaptcha:     Similar to Turnstile but shows puzzles — worse UX.
reCAPTCHA v2: Puzzles, Google data collection — privacy concern for DPDP compliance.
reCAPTCHA v3: Invisible but opaque scoring; sends user behaviour data to Google.
For a premium-positioned Indian product: Turnstile is the right call.
Cost: ₹0. Setup: 10 minutes.

setStatus(res.ok ? 'success' : 'error')
}
return (
<form onSubmit={handleSubmit}>
{/* ... form fields ... */}
<TurnstileWidget
onVerify={(token) => setCfToken(token)}
onExpire={() => setCfToken(null)}
/>
<button type='submit' disabled={!cfToken || status === 'loading'}>
{status === 'loading' ? 'Joining...' : 'Join Waitlist'}
</button>
</form>
)
}

// lib/turnstile.ts
export async function verifyTurnstile(token: string): Promise<boolean> {
const res = await fetch(
'https://challenges.cloudflare.com/turnstile/v0/siteverify',
{
method: 'POST',
headers: { 'Content-Type': 'application/json' },
body: JSON.stringify({
secret:   process.env.TURNSTILE_SECRET_KEY,
response: token,
}),
}
)
const data = await res.json() as { success: boolean }
return data.success
}

Stage 1: Fast gates    — lint, typecheck, build        (~90s)   BLOCKING

Stage 4: Performance   — Lighthouse CI: mobile ≥90     (~120s)  BLOCKING
Total wall time: ~6 minutes. Every PR is safe to merge only if all 4 pass.
Cost: GitHub Actions free tier = 2,000 minutes/month (more than enough).
